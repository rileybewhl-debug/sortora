How we protect your business
Multiple security systems work together to keep your revenue, your customers, and your data safe.

Payment data never touches our servers

When a participant pays their share, they're redirected to a Stripe-hosted checkout page. Card details go directly to Stripe's PCI Level 1 certified servers — the highest level in the payments industry.

Sortora only receives a webhook confirmation that payment succeeded. We never see, store, or process card numbers, CVVs, or bank details.

Tamper-proof by design

The Sortora widget is a lightweight form that runs inside a Shadow DOM — a browser-native isolation boundary. The host website can't access the widget's internals, and the widget can't access the host page's data.

All logic runs server-side. Even if someone modifies the widget JavaScript, the server enforces platform fees, validates inputs, and checks subscription status on every request.

CSS and JS boundaries prevent cross-contamination

Enterprise-grade infrastructure

Sortora runs on Vercel's global edge network with automatic SSL provisioning and DDoS protection. Data is stored in Supabase with Row Level Security policies, encrypted connections, and automated backups.

Passwords are hashed with bcrypt using per-password salts. Session tokens are short-lived JWTs validated on every request. Service keys exist only in server-side environment variables — never in client code.

Built for trust

Webhook verification

Every incoming Stripe webhook is verified against a cryptographic signing secret. Only Stripe can trigger payment confirmations — not an attacker.

HSTS enforcement

Strict-Transport-Security headers ensure connections can never be downgraded to unencrypted HTTP. Every request uses HTTPS — no exceptions.

No error leakage

Server errors return generic messages to the browser. Internal details, stack traces, and database errors are logged privately and never exposed to users.

Subscription gating

The widget checks subscription status on every load. Expired plans, disconnected Stripe accounts, and limit overages are blocked server-side — not on the frontend.

Compliant infrastructure

Sortora is built on industry-standard infrastructure from Stripe, Vercel, and Supabase — all of which maintain SOC 2 compliance and undergo regular third-party security audits.
